Reading Ch10: Testing and Debugging

  • System Errors - not much influence here, but be aware of issues
  • Programmer Errors - incorrect syntax or logic...try to predict as much as possible to ensure good user experience
  • User Error - usually incorrect data entered by user, BUT we should anticpate and design in a way to prevent theses issues
  • Fail Loudly - make sure exceptions are caught so you can deal with them and fix code...this is easier and more ideal than silent errors that taker forever to pinpoint.
  • Using Strict Mode
    • produces more exceptions/warnings; prohibits use of deprecated features
    • Helps to improve code clarity and speed
    • 'use strict' is added to first line of JavaScript file, but recommended to use in self-invoking fucntions (especially if code sharing)
    • modules are strict mode by default
  • Feature Detection to ensure browser supported features (use Modernizr or Can I Use for resources)
  • Browser Debugging and Console Work
    • use breakpoints and alert() method outdated
    • get familiar with broweser built in dev tools
  • Test-Driven Development - "any tests are better than no test al all"
  • consider usuing a TDD framework like Jest
